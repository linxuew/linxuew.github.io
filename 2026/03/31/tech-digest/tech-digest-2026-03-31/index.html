无线与路由技术动态 2026-03-31

发表于2026-03-31|更新于2026-03-31|技术动态
|阅读量:

本期摘要

Linux 7.0-rc6 进入收尾阶段，Linus 对补丁量略感不安但仍预计本周末 RC 终结；OpenWrt 社区围绕 25.12.x 升级引发的 bonding 兼容性问题与多款设备变砖事件持续活跃，值得近期准备升级的用户重点关注。

Linux 动态

Kernel 7.0-rc6 发布，Linus 预计下周 RC 收官

Linux 7.0-rc6 正式放出测试版本。Linus 坦言本轮 RC 补丁量”比往常略多”，但整体走势并未令其决定延期发布，目前仍预期下周末发布最终 RC。开发者和下游发行版维护者可保持关注并积极提交回归报告。

🔗 原文

SystemRescue 13.00 发布，内置 6.12.20 LTS 内核

基于 Arch Linux 的系统救援发行版 SystemRescue 发布 13.00 版本，内核升级至 6.12.20 LTS，同时将 bcachefs 工具链及内核模块更新至 1.37.3，并同步升级众多软件包。对于需要在生产环境中进行磁盘恢复或 bcachefs 文件系统运维的工程师，这是一次值得更新的版本。

🔗 原文

Rspamd 4.0.0 发布，新增 HTML 模糊钓鱼检测

反垃圾邮件框架 Rspamd 发布 4.0 大版本，核心新特性包括 HTML 模糊钓鱼检测（HTML fuzzy phishing detection）以及模糊哈希最多支持八个标志位。该版本含有 breaking changes，使用自定义规则或插件的管理员在升级前应仔细阅读 changelog。

🔗 原文

Rust 下一代 Trait Solver 进展解析

LWN 深度报道了 Rust 编译器团队正在推进的 trait solver 重写项目。新的求解器旨在简化未来语言特性的实现路径，并修复长期以来难以解决的边缘场景 bug。对于在嵌入式或路由器固件中使用 Rust 开发的工程师，这一编译器底层演进值得持续跟踪。

🔗 原文

无线通信动态

Blue Origin “Project Sunrise”：轨道数据中心与 Starlink 的差异化竞争

Frost & Sullivan 分析师指出，Blue Origin 发布大规模卫星星座计划的动机并非单纯追赶 Starlink，而是试图在轨道数据中心赛道上”圈地”。这一战略意味着低轨道算力基础设施正成为卫星互联网竞争的新维度，超越了单纯的宽带接入市场。

🔗 原文

意大利邮政 125 亿美元竞购 TIM，折射欧洲数字主权焦虑

Omdia 分析认为，意大利邮政（Poste Italiane）对电信运营商 TIM 发起的 125 亿美元收购要约，是欧洲各国政府将关键网络基础设施纳入国家控制的系统性行动之一。欧洲电信整合浪潮中，”主权”已成为超越商业逻辑的核心叙事。

🔗 原文

HPE 在 MWC 强调 AI 时代光纤与长途网络的中期升级需求

HPE 在 MWC 上重申：AI 流量爆发正倒逼运营商对光纤互联与长途骨干网进行一次”中代际升级”，其 PTX 和 MX 产品线分别承接云端与边缘推理两大场景。边缘推理侧需求尚处早期，但产品路线图已清晰落地。

🔗 原文

TELUS 与 P.I. Works 携手：Open RAN 时代的 AI 网络自动化实践

TELUS 与 P.I. Works 联合展示了基于 EVO 平台的 AI 驱动网络自动化方案，该方案运行于硬件之上的软件智能层，可针对 5G Standalone 网络实现自动化管理与性能优化。相比 Open RAN 未来愿景，这一方案已在商用网络中落地，提供了可量化的工程价值参考。

🔗 原文

WiFi 标准 & 技术

Wi-Fi 网关正演变为边缘 AI 集成平台（Qualcomm 视角）

RCR Wireless 报道，现代 Wi-Fi 网关的系统架构正从单一无线接入设备向”高性能无线连接 + 片上算力 + AI 推理”的集成边缘平台演进。Qualcomm 是推动这一趋势的核心力量之一，其网关芯片已开始整合本地 AI 能力以支撑低延迟应用场景。这一趋势对 OpenWrt 等开源固件生态的硬件适配提出了新要求。

🔗 原文

路由器 & OpenWrt

OpenWrt 25.12.x 中 bonding 的 arp_interval 兼容性问题

升级至 OpenWrt 25.12.x 后，部分用户在使用 802.3ad（LACP）bonding 模式时遭遇启动报错：arp_interval: mode dependency failed, not supported in mode 802.3ad(4)。移除 monitor_mode 'arp' 配置后 bond 接口无法正常建立，问题根源在于该版本对 bonding 参数兼容性校验逻辑的变更。依赖链路聚合的用户升级前应先确认配置兼容性。

🔗 原文

WNR3500L V1 升级 25.12.2 后变砖，TFTP 救砖记录

用户从 OpenWrt 25.12.0 升级至 25.12.2 后，Netgear WNR3500L V1 出现变砖。帖子详细记录了在 Ubuntu 上配置 TFTP 服务的步骤与排查过程，为同款设备用户提供了可复现的救砖参考。25.12.x 系列对部分老旧 MIPS 设备的兼容性问题已多次被社区报告，建议升级前备份并确认设备支持状态。

🔗 原文

ZTE F601 V9 GPON SN 疑存储于芯片内部而非 Flash

一位用户在获取 ZTE F601 V9 的 GPON 序列号时，遍历全部 MTD 分区均未找到 ZTEGxxxx 格式的序列号，推测 SN 存储于 GPON 芯片内部寄存器而非外部 Flash。这一发现对需要迁移或克隆光猫序列号的 DIY 用户具有重要参考价值，也再次揭示了运营商定制固件的硬件锁定机制。

🔗 原文

Unifi U6+ 刷回原厂后再刷 OpenWrt 遭遇哈希校验失败

用户在将 Unifi U6+ 通过 TFTP 恢复至原厂固件后，尝试再次刷入 OpenWrt 时遭遇 Failed hash integrity / can't load kernel 错误导致启动循环。该问题与 Unifi 固件对 flash 分区签名验证机制相关，社区正在排查具体原因。有意在 U6+ 上玩机的用户需留意此风险。

🔗 原文

小米 AX3200 在 OpenWrt 下 Wi-Fi 5 速率被限制的排查思路

用户反映小米 AX3200 在 OpenWrt 下 Wi-Fi 5 实测最高仅 200 Mbps，远低于设备标称的 433 Mbps，而有线连接速率正常。此类问题通常与信道宽度配置（20/40/80 MHz）、发射功率限制或驱动默认参数有关，社区建议逐一核查无线接口的 htmode 和 txpower 设置。

🔗 原文

编辑小结

本期最值得关注的主线是 OpenWrt 25.12.x 的升级稳定性问题：bonding 兼容性报错、多款设备变砖、OWUT 镜像下载失败在同一天集中出现，建议持观望态度的用户暂缓升级，等待 25.12.3 或社区明确 fix 后再行操作。Wi-Fi 网关向边缘 AI 平台演进的趋势已从概念走向 Qualcomm 的具体产品路线，这对 OpenWrt 社区的驱动适配和包管理提出了新挑战。Linux 7.0 内核预计数日内 RC 收官，路由器固件和嵌入式发行版的维护者可提前做好跟进准备。
