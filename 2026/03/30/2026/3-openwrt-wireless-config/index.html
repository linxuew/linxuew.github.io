OpenWrt WiFi栈全貌

OpenWrt 无线子系统从上到下分为六层，各层职责明确、接口固定。wifi scripts 是控制器，负责生成配置并启动进程；hostapd/wpa_supplicant 是独立的用户态 daemon，两者之间有明确的生成配置→拉起进程的层次关系。

各层职责：

  • 应用层：UCI 文件、LuCI、自定义脚本，通过 uci 命令或 ubus RPC 写入配置
  • netifd：网络管理守护进程，解析 UCI，维护接口状态机，调度 wifi scripts；同时通过 ubus 被动感知事件，同步接口状态
  • wifi scripts（控制器）：mac80211.sh / hostapd.sh 等，生成 hostapd.conf 并通过 procd 拉起 hostapd 进程
  • hostapd / wpa_supplicant（执行器）：独立用户态 daemon，AP 认证、802.11i/r/k/v、WPS、EasyMesh；通过 nl80211 netlink API 与 cfg80211 通信
  • cfg80211 / nl80211：内核统一无线接口，管理 channel、txpower、scan、regulatory（国家码），屏蔽驱动差异
  • 驱动层（三类）：MTK 商用 mt_wifi（闭源 vendor SDK）、SoftMAC（mt76/ath9k，完整走 mac80211）、FullMAC（ath10k/ath11k mac80211控制+firmware执行MAC；brcmfmac cfg80211 thin wrap；wl 私有 ioctl 旁路 cfg80211）

事件路径分两条并行通道：路径 A 经 driver → cfg80211 → nl80211 → hostapd → ubus → netifd 状态同步 + app 订阅；路径 B 由 cfg80211 → nl80211 multicast 直接到 iw event / wpa_cli，不经过 hostapd。

驱动层分类

驱动层分三类，选型直接影响调试命令和运行时配置接口：

类型 代表驱动 与 mac80211 关系 配置接口
MTK 商用（闭源） mt_wifi（MT7986/MT7987/MT7990） cfg80211 注册，但大量通过私有 nl80211 vendor cmd 下发 iwpriv ra0 / mtwifi_dbg
SoftMAC（开源） mt76（社区版）/ ath9k 驱动调用 mac80211，完整走标准协议栈 iw / debugfs
FullMAC ath10k / ath11k mac80211 作控制层，firmware 执行大量 MAC 逻辑 iw（有限）/ vendor cmd
私有接口 brcmfmac / wl brcmfmac 是 cfg80211 thin wrap；wl 私有 ioctl 旁路 cfg80211 wl assoclist / wl bss info

MT7990 商用平台使用 mt_wifi，不是开源 mt76。两者 debugfs 路径、iwpriv 命令、txbf 控制方式完全不同，不可混用。

netifd 的调度职责

netifd 是 OpenWrt 网络管理守护进程，跑在 uloop 单线程事件循环上，核心职责：

  • 解析 config wifi-device / config wifi-iface，转成运行时对象
  • 维护接口状态机：idle → pending → connecting → connected
  • 通过 ubus 暴露 network.wireless、network.interface.* RPC 对象
  • 调用 /lib/netifd/wireless/mac80211.sh 驱动 hostapd 生命周期（配置路径）
  • 被动感知事件：订阅 hostapd 通过 ubus 上报的状态变更，同步接口 up/down 状态后对外广播（事件路径 A）

/lib/netifd/ 关键文件：

文件 作用

proto/dhcp.sh 调用 udhcpc，汇报 IP 给 netifd

配置变更的三个级别

不同参数刷新代价差异极大，必须按级别选择最小代价路径。

级别 触发参数 路径 STA 断线
PHY 级 channel · htmode · country · noscan uci commit + wifi reload 是，全部
BSS 级（持久） ssid · encryption · ieee80211r uci commit + hostapd_cli reload 否
BSS 级（临时） wpa_passphrase · ap_isolate · max_num_sta hostapd_cli set 否
运行时 txpower · txbf · bitrate mask iw / debugfs 否

事件与通知

无线事件有两条并行的上报路径，不可混淆：

  • 路径 A（经 hostapd）：driver → cfg80211 → nl80211 event → hostapd → ubus notify。netifd 订阅 ubus 被动感知状态变更（vif up/down），完成自身状态机同步后再对外广播；app 同样通过 ubus 订阅归一化后的事件。
  • 路径 B（直接）：cfg80211 → nl80211 multicast group → iw event / wpa_cli。不经过 hostapd，适合低延迟诊断工具或底层守护进程直接订阅。
总线 / 路径 订阅方式 适用场景
hostapd ctrl_iface（路径 A） hostapd_cli -i wlan0 -a script STA 上下线，含 MAC/reason，最及时
ubus 事件（路径 A 汇总） ubus listen network.interface.* netifd 归一化后的 ifup/ifdown，含 IP
iw event（路径 B） iw event nl80211 multicast，不经 hostapd，诊断用
内核 netlink（路径 B） nl_socket_add_membership MCGRP_MLME MLME 原始事件，适合 C 守护进程
